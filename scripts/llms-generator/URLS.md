# llms.txt URL scheme

Base origin: `https://gravity-ui.com`

Generated by `scripts/llms-generator/` (entry: `scripts/generate-llms.mjs`).

## Root files

| URL              | Contents                                                                               |
| ---------------- | -------------------------------------------------------------------------------------- |
| `/llms.txt`      | Catalog of all libraries — positioning + `When to use` / `When not to use` per package |
| `/llms-full.txt` | Aggregate: root + every per-package file of every version                              |

## Per-package (three address levels)

For each library `<id>`:

| URL                             | Resolves to                                      |
| ------------------------------- | ------------------------------------------------ |
| `/llms/<id>/llms.txt`           | **latest** version (stable alias)                |
| `/llms/<id>/<major>/llms.txt`   | latest generated version **within a major line** |
| `/llms/<id>/<version>/llms.txt` | a **specific** version                           |

## Concrete examples

**uikit**

```
https://gravity-ui.com/llms/uikit/llms.txt          → latest (7.46.0)
https://gravity-ui.com/llms/uikit/7/llms.txt        → latest of major 7 (7.46.0)
https://gravity-ui.com/llms/uikit/6/llms.txt        → latest of major 6 (6.43.4)
https://gravity-ui.com/llms/uikit/7.46.0/llms.txt   → exact version
https://gravity-ui.com/llms/uikit/7.45.0/llms.txt   → exact version
https://gravity-ui.com/llms/uikit/6.43.4/llms.txt   → exact version
```

**aikit**

```
https://gravity-ui.com/llms/aikit/llms.txt          → latest
https://gravity-ui.com/llms/aikit/2/llms.txt        → latest of major 2 (2.11.0)
https://gravity-ui.com/llms/aikit/2.11.0/llms.txt   → exact version
```

## Which versions are generated (per package)

- **All patches of the latest minor** (e.g. `7.46.0`).
- **The `.0` of each of the last `LAST_MINORS` (10) minor tracks** (`7.45.0`, `7.44.0`, …).
- **The latest version of each of the last `LAST_MAJORS` (2) major tracks** (e.g. `6.43.4`).
- Plus the alias files (`<id>/`, `<id>/<major>/`) mirroring each such version.

Tunable in `constants.mjs`: `LAST_MINORS = 10`, `LAST_MAJORS = 2`.

## Per-file content

Every per-package file starts with a banner:

```
> Generated for `@gravity-ui/<id>@<version>`
```

Then the body is chosen as follows:

- If the version has a published `INDEX.md` on unpkg (`build/docs/INDEX.md` or
  `dist/docs/INDEX.md`) → its content is used **verbatim**, with the concrete
  version substituted for the `**installed**` placeholder.
- Otherwise the page is **assembled** from the README's `## For AI agents`
  section (positioning, install, usage, agent block, links).

## Filesystem layout

```
public/
  llms.txt                  # root catalog
  llms-full.txt             # aggregate
  llms/
    <id>/
      llms.txt              # latest alias
      <major>/
        llms.txt            # per-major alias
      <version>/
        llms.txt            # per-version file
```
