Su equipo de servicio puede usar el registro de cambios para informar a los usuarios sobre los cambios.

El registro de cambios se muestra una vez cuando el usuario abre la página de servicio. No ofrecemos la opción de volver a abrir la página de registro de cambios. Impleméntelo de forma independiente si es necesario.

Como el registro de cambios se muestra en una ventana modal, sigue las reglas para cerrar y escalar las ventanas modales.

## Elementos

El registro de cambios es una ventana modal de tamaño M con publicaciones ordenadas cronológicamente. Cada publicación tiene:

1. Fecha de publicación
2. [Etiqueta](https://gravity-ui.com/design/guides/label) «Nueva» (opcional)
3. Encabezado de publicación
4. Publicar imagen (opcional)
5. Publicar texto
6. [Botón](https://gravity-ui.com/design/guides/button) de acción (opcional).

![Elements](/static/images/design/ChangelogDialog/ChangelogDialog1.png)

### Fecha de publicación

A partir de la fecha de publicación, el usuario puede comprender cuándo se produjo un cambio en particular en el servicio. De forma predeterminada, la fecha tiene el formato DD Mon YYYY. También puede usar otros formatos de fecha para que coincidan con la configuración regional del servicio.

### Etiqueta

Usa la etiqueta «Nuevo» para las publicaciones nuevas. El servicio determina qué publicaciones considerar nuevas y marcarlas con esta etiqueta. En el texto, puede usar otras palabras, diferentes de «Nuevo».

### Encabezado de publicación

El encabezado de la publicación se usa para una breve descripción del cambio de servicio. El encabezado puede tener cualquier longitud. No recomendamos:

- Utilizar encabezados que ocupen más de una línea (que contengan más de 70 caracteres), ya que esto empeora la legibilidad del texto.
- Para copiar el texto de las noticias al encabezado.

El encabezado usa el estilo text-subheader-3. No proporcionamos otros estilos de texto para los encabezados.

### Imagen

La imagen es opcional y tiene unas dimensiones fijas de 516x240 px. Puede utilizar archivos JPG y PNG como imágenes (no se admiten imágenes GIF ni archivos de vídeo). Usa imágenes cuando necesites ilustrar el tema de la publicación. Se usa una imagen para los temas de interfaz claros y oscuros.

Si subes una imagen de más de 516 x 240 píxeles, se escalará y se centrará en el lado más grande.

Teniendo en cuenta las pantallas Retina, recomendamos subir imágenes con una escala doble en cada lado, es decir, de 1032 x 480 píxeles.

### Publicar texto

El texto de la publicación sigue a la imagen y puede tener cualquier longitud. No recomendamos:

- Utilizar textos muy largos (más de 500 caracteres). Si no puede acortar el texto a la longitud recomendada, puede agregar enlaces a la documentación dentro del texto o incluir un botón después del texto para navegar hasta el texto completo.
- Para crear subencabezados. Dado que la longitud de texto recomendada es concisa, no es necesario dividir el texto en secciones mediante subtítulos.

El texto usa el estilo 'text-body-1'. No proporcionamos otros estilos de texto, pero puede añadir formato, es decir, crear listas numeradas y con viñetas, tachar, subrayar, negrita y cursiva.

### Botón de acción

El botón de acción es opcional y se encuentra debajo del texto de la publicación. Se usa para navegar hasta el texto completo de la publicación.

Solo se admiten botones de tamaño M. El botón puede ser de cualquier tipo, con o sin icono. Cíñete a un solo tipo de botón.

No admitimos varios botones.

### Otros elementos

Además, en el encabezado de la ventana modal, puede agregar un enlace «Ver registro de cambios completo» a una página de registro de cambios (puede ser una página de servicio independiente, un libro de cuentos o un servicio de documentación externo).

## Recomendaciones de uso

Usa el registro de cambios:

- Cuando necesite informar al usuario sobre cambios significativos en el servicio.
- Cuando necesite mantener al usuario al tanto de cambios menores. En este caso, considere la posibilidad de mostrar el registro de cambios periódicamente e informar sobre cualquier cambio menor desde la última entrada del registro de cambios.
- Cuando necesite recordar al usuario los cambios en el servicio y aumentar su conocimiento de las versiones anteriores.

No recomendamos:

- Para usar el registro de cambios para incorporar usuarios.
- Utilizar el registro de cambios para informar al usuario sobre cada cambio menor en el servicio.
