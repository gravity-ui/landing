Un componente para seleccionar un valor o un conjunto de valores de una lista especificada. Cuando el usuario hace clic en el control, se abre una lista desplegable con un conjunto de valores para que el usuario elija.

## Casos de uso

![1.png](/static/images/design/Select/select-0001.png)

**¿Cuándo usar?**

- Cuando espera que el usuario seleccione valores de una lista especificada.

**¿Cuándo evitar?**

- Cuando quiere mostrar todos los valores para seleccionar. Para tales casos de uso, utilice [Radio](/design/guides/radio), [Checkbox](/design/guides/checkbox), o Radio Button.

## Opciones

**Selección única**

![2.png](/static/images/design/Select/select-0002.png)

- Utilice para seleccionar un solo valor de una lista.
- Después de que el usuario selecciona un valor, la lista desplegable se cierra.
- El control muestra el valor seleccionado.

**Selección múltiple**

![3.png](/static/images/design/Select/select-0003.png)

- Utilice cuando espera que el usuario seleccione más de un valor de una lista.
- En la lista, el valor seleccionado está marcado con una marca de verificación a la izquierda.
- El control muestra los valores seleccionados separados por comas.
- Para cerrar la lista desplegable y aplicar los valores seleccionados, el usuario hace clic en el espacio en blanco fuera o en el botón de confirmación en la lista (si se proporciona).

## Tipos

El control viene en dos tipos: **Normal** y **Clear**, ambos tienen el mismo conjunto de funciones pero se ven diferentes y se aplican dependiendo del contexto.

![4.png](/static/images/design/Select/select-0004.png)

**Normal**

- El tipo más frecuentemente utilizado.
- La característica distintiva es que el componente tiene un borde de 1 px, y los elementos dentro (texto y chevron) tienen márgenes desde los bordes.
- Utilice este componente en formularios donde necesite separar elementos del formulario usando márgenes.

**Clear**

- La característica distintiva es que el componente no tiene borde, y los elementos dentro (texto y chevron) están al ras con los bordes del campo de entrada.
- Utilícelo cuando el tipo Normal crea ruido visual.
- Al usar este tipo, asegúrese de que el significado del componente siga siendo claro para el usuario.

## Estructura

![5.png](/static/images/design/Select/select-0005.png)

1. Etiqueta en línea
2. Valor
3. Icono de borrar
4. Chevron
5. Icono de error
6. Descripción del error

### Etiqueta en línea

   ![6.png](/static/images/design/Select/select-0006.png)

   - Un elemento opcional.
   - Añade una explicación al nombre del campo select.
   - Puede reemplazar la etiqueta externa en el formulario.

### Valor

   ![7.png](/static/images/design/Select/select-0007.png)

   - Un elemento obligatorio.
   - Por defecto, es una pista para ayudar al usuario a entender qué se puede seleccionar de la lista de valores.
   - El lugar donde se muestra el valor o valores seleccionados.

### Icono de borrar

   ![8.png](/static/images/design/Select/select-0008.png)

   - Un elemento opcional.
   - Utilícelo para borrar rápidamente el componente de valores seleccionados u opciones de valor sin invocar la lista desplegable.
   - Puede aparecer cuando se ha seleccionado al menos un valor.
   - Después de borrar, el foco permanece en el componente.
   - Cambia de color al pasar el cursor.

### Chevron

   ![9.png](/static/images/design/Select/select-0009.png)

   - Un elemento obligatorio.
   - Utilícelo para diferenciar el componente de otros componentes visualmente similares (por ejemplo, [TextInput](/design/guides/text-input)), para identificar el componente en el formulario.
   - El componente puede estar solo con un chevron, por ejemplo, cuando necesita una vista compacta.
   - A pesar de la presencia de un chevron, toda el área del componente es clickeable.

### Icono de error

   ![10.png](/static/images/design/Select/select-0010.png)

   - Se muestra solo en el estado Inline Error
   - Funciona junto con un Tooltip de descripción de error que se muestra al pasar el cursor sobre el icono. Ayuda a mantener al usuario informado pero ahorra espacio.

### Descripción del error

   ![11.png](/static/images/design/Select/select-0011.png)

   - Aparece solo en el estado Outline Error.
   - La descripción del error se coloca en la parte inferior izquierda del campo cuando necesita mostrar inmediatamente la causa del error.

## Estados

Tiene varios estados: **Suggest, Default, Hover, Disabled, Active, Inline Error, Outline Error.**

![12.png](/static/images/design/Select/select-0012.png)

1. **Suggest.** Estado inactivo, cuando el usuario no está interactuando directamente con el componente y no ha seleccionado nada en él. En este estado, puede dar una pista al usuario sobre lo que puede seleccionar.
2. **Default.** Estado inactivo, cuando el usuario no está interactuando directamente con el componente y ya ha seleccionado algo en él. En este estado, el valor de la pista cambia a la selección de la lista desplegable.
3. **Hover.** Estado activo cuando el usuario mueve el cursor sobre el área del componente.
4. **Disabled.** Estado inactivo del componente cuando el usuario no puede interactuar con él.
5. **Active.** Estado activo habilitado cuando el usuario ha hecho clic en el control. Una lista desplegable se abre al hacer clic.
6. **Outline error.** Un estado del componente activado por un error de validación, que se muestra en la parte inferior del campo con una explicación de lo que ha sucedido.
7. **Inline error.** Un estado del componente activado por un error de validación, pero la explicación del error no aparece, solo un icono especial. Cuando pasa el cursor sobre el icono, aparece un Tooltip con el error.

## Tamaños

![13.png](/static/images/design/Select/select-0013.png)

Hay cuatro tamaños de componente: **S, M, L, XL**.

| Tamaño | Altura | Estilo de texto | Redondeo |
| --- | --- | --- | --- |
| S | 24 | Body 1 | 5 |
| M | 28 | Body 1 | 6 |
| L | 36 | Body 1 | 8 |
| XL | 44 | Body 2 | 10 |

La elección del tamaño determina la altura y las esquinas redondeadas del control, sus rellenos y el tamaño del texto.

El radio de las esquinas depende del tamaño y se establece mediante variables.

1. **S.** Utilícelo donde el control estándar es demasiado grande (tablas, tarjetas pequeñas).
2. **M.** Este es el tamaño principal, utilícelo en la mayoría de los formularios y filtros.
3. **L.** Se usa muy raramente.
4. **XL.** Utilícelo en entornos de comunicación, es decir, sitios web y páginas de destino.

## Recomendaciones de uso

1. **Trabajando con el campo**
   - Elija el **ancho del campo** dependiendo del tamaño del contenido. Por ejemplo, puede hacer el elemento más ancho para valores largos o selección múltiple.
   - Use **un tamaño** con otros componentes complementarios en el formulario. Si eligió trabajar con M, los otros componentes deberían ser del mismo tamaño.
   - Use el **icono de cruz para borrar** la selección múltiple: esto es más conveniente y rápido.
   - No puede cambiar la altura del control. Solo proporcionamos el **tamaño de campo fijo**.

2. **Etiqueta de contorno**

   ![14.png](/static/images/design/Select/select-0014.png)

   - Un nombre de campo breve (no debe exceder una línea).
   - Cuando lo use en formularios, alinéelo con el valor dentro del campo.

3. **Pista con un tooltip**

   ![15.png](/static/images/design/Select/select-0015.png)

   - Pista adicional sobre la elección, mostrada al pasar el cursor sobre el icono de signo de interrogación.
   - Utilícela cuando necesite explicar la etiqueta.

4. **Descripción**

   ![16.png](/static/images/design/Select/select-0016.png)

   - Puede estar ubicada debajo del campo o a la derecha del campo.
   - Utilícela en casos especiales para hacer aclaraciones adicionales sobre la elección o dar explicaciones para los valores elegidos.
