## Structure

![IMG - 18.png](/static/images/design/Paginator/paginator-0005.png)

1. **Current page.** Shows the serial number of the displayed page.
2. **Previous page.** Opens the previous page.
3. **Next page.** Opens the next page.
4. **Last page of content.** Shows up if the total content is known and can be divided into pages right away. Opens the last page by a click.
5. **Overflow icon.** Shows up if the number of pages exceeds four.
6. **Forward/backward switches.** Switch pages one by one.
7. **Go to the first page.** Opens the first content page.
8. **Go to a specific page.** Opens the specified page when the user enters the page number and presses Enter.
9. **Number of items per page.**  A switch for the number of elements displayed on one page.

You do not have to use all elements of the structure together. You can find the available configurations under "Modifications".

## Modifications

![IMG - 20.png](/static/images/design/Paginator/paginator-0008.png)

1. **Page Size Options and Show Input**: You can disable these fields if the user cannot go to a specific page or control the amount of content displayed on the page.
2. **Total Unknown** only displays the current page. Use when the total number of elements is unknown
3. **Hide Pages** ensures page switching without serial numbers. Use when the list of elements is dynamic and the number of pages, as well as the serial number, will constantly change and confuse the user.
4. **Compact** is a compact format without text. Use when the interface space is limited (for example, in dialog boxes)

For mobile device interfaces, you can use the mobile version of the paginator.
