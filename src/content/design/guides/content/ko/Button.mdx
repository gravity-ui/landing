버튼은 사용자 인터페이스의 상호작용 요소로, 클릭 시 폼 제출, 변경 사항 저장, 다음 단계로 이동 등과 같은 작업을 수행합니다. 다른 페이지나 웹사이트로 이동해야 하는 경우 링크를 사용하세요.

## 모양

버튼에는 액센트, 기본, 의미론적, 돌출, 대비의 다섯 가지 유형이 있습니다.

### **액센트**

![Accent](/static/images/design/Button/Button1.png)

액센트 버튼은 주요 작업을 강조하고 중요한 작업에 사용자 주의를 집중시킵니다. 액션, 외곽선 액션, 플랫 액션의 세 가지 주요 스타일이 있습니다.

인터페이스에서 작업에 부여하려는 강조 정도에 따라 이러한 액센트 버튼 유형 중에서 선택하세요. 가장 높은 우선순위에는 액션을, 중간 우선순위에는 외곽선 액션을, 낮은 우선순위 또는 보조 작업에는 플랫 액션을 사용하세요.

인터페이스에서 액센트 버튼은 절제하여 사용하고, 가능하다면 페이지당 하나로 제한하세요(대화 상자 제외). 이렇게 하면 사용자에게 주요 작업을 강조하고 중요성을 부각할 수 있지만, 너무 많은 액센트 요소로 인터페이스가 과부하되는 위험을 완화할 수 있습니다.

### 기본

![Primary](/static/images/design/Button/Button2.png)

기본 버튼은 인터페이스의 주요하고 자주 사용되는 작업을 수행합니다. 이러한 버튼에는 일반, 외곽선, 플랫, 플랫 보조의 여러 스타일이 있습니다. 인터페이스에서 특정 작업에 부여하려는 강조 정도에 따라 이러한 스타일 중에서 선택하세요. 예를 들어, 일반 버튼은 표준 작업에 적합하고, 외곽선 버튼은 중간 우선순위 작업에 적합하며, 플랫 및 플랫 보조 버튼은 덜 중요하거나 보조 작업용입니다.

### 의미론적

의미론적 버튼은 특정 컨텍스트 또는 작업을 전달하여 사용자에게 의미를 부여합니다. 이를 사용하여 사용자가 수행되는 작업과 그 의미를 쉽게 이해하도록 돕습니다.

의미론적 버튼에는 정보, 성공, 경고, 위험의 네 가지 주요 스타일이 있습니다.

### 정보

![Info](/static/images/design/Button/Button3.png)

추가 정보, 힌트 또는 컨텍스트를 얻는 작업과 관련된 경우 사용하세요. 정보 버튼은 다른 리소스로 이동할 옵션을 강조해야 할 때 하이퍼링크로도 사용할 수 있습니다. 예: 추가 정보, 문서, 도움말.

**성공**

![Success](/static/images/design/Button/Button4.png)

성공적으로 완료된 작업을 강조하는 데 사용하세요.

**경고**

![Warning](/static/images/design/Button/Button5.png)

잠재적 위험 또는 경고로 인해 사용자 주의가 필요한 작업을 강조하는 데 사용하세요. 이러한 버튼은 사용자가 작업의 가능한 결과를 이해하도록 돕고 주의를 기울이도록 권장합니다.

**위험**

![Danger](/static/images/design/Button/Button6.png)

파괴적이거나 부정적인 결과를 초래할 수 있는 작업을 강조하는 데 사용하세요. 잠재적인 부정적인 측면(예: 삭제, 중지, 연결 해제)을 강조해야 하는 경우에만 신중하게 사용하세요.

**돌출**

![Raised](/static/images/design/Button/Button7.png)

돌출 버튼은 콘텐츠 위로 튀어나오는 버튼 유형입니다. 화면의 중요한 작업을 강조하는 데 사용하며, 일반적으로 고정된 위치에 있습니다. 이 유형의 버튼은 사용자가 즉시 알아차려야 하는 주요 작업에 적합합니다.

이러한 버튼은 지도 인터페이스 및 이미지 기반 애플리케이션에서도 확대/축소, 이동 또는 이미지 데이터 조작과 같은 다양한 작업에 자주 사용됩니다.

**대비 버튼**

![Contrast](/static/images/design/Button/Button8.png)

**대비 버튼**은 복잡한 배경에서 작업을 강조하는 데 사용되는 특수 버튼 유형입니다. 밝고 명확한 대비를 만들어 배너 또는 마케팅 자료와 같이 다양하고 채도가 높은 배경에서도 뛰어난 가시성을 제공합니다.

## 구조

인터페이스의 버튼 구조에는 다양한 디자인 옵션이 포함됩니다.

![Anatomy](/static/images/design/Button/Button9.png)

1. **텍스트만**: 텍스트만 포함하는 버튼입니다. 텍스트가 버튼의 목적을 명확하고 모호하지 않게 전달할 때 유용하며, 특히 텍스트가 짧고 유익할 때 유용합니다. 예를 들어, 생성 또는 취소와 같은 작업에 이러한 버튼을 사용할 수 있습니다.
2. **오른쪽 아이콘**: 아이콘이 텍스트 오른쪽에 배치된 버튼입니다. 이 옵션은 동일한 작업을 수행하는 다른 방법을 제시하는 데 유용합니다. 예를 들어, 사용자가 여러 옵션(PDF, RTF, HTML로 저장) 중에서 선택할 수 있는 저장 작업에 이러한 버튼을 사용하세요.
3. **왼쪽 아이콘**: 아이콘이 텍스트 왼쪽에 배치된 버튼입니다. 아이콘이 중요한 정보를 전달하고 강조해야 할 때 이 옵션을 사용하세요. 예를 들어, 시작 또는 삭제 작업에 이러한 버튼을 사용할 수 있습니다.
4. **왼쪽 및 오른쪽 아이콘**: 텍스트 양쪽에 아이콘이 배치된 버튼입니다. 아이콘으로 주요 작업을 강조하고 실행을 위한 다양한 경로를 제공하려면 이 옵션을 사용하세요. 예를 들어, 사용자가 여러 옵션(프로젝트, 리소스, 문서 추가) 중에서 선택할 수 있는 추가 작업에 이러한 버튼을 사용할 수 있습니다.
5. **아이콘만**: 텍스트가 없고 아이콘으로만 구성된 버튼입니다. 버튼의 정보가 시각적으로 이해되고 추가 텍스트 설명이 필요하지 않을 때 이 옵션을 사용하세요. 또한 텍스트 레이블을 위한 공간이 없는 제한된 공간에서 여러 작업을 간결하게 그룹화하는 데 사용할 수도 있습니다. 예를 들어, 항목 목록에서 삭제 버튼입니다. 이 경우 텍스트 레이블 없이 휴지통 아이콘으로 삭제 옵션을 제공할 수 있습니다.

## 상태

인터페이스의 모든 버튼에는 사용자의 버튼 가용성 및 현재 작업을 시각적으로 나타내는 다섯 가지 기본 상태가 있습니다.

![States](/static/images/design/Button/Button10.png)

**기본**: 상호작용을 위해 준비되어 있으며 사용자 작업을 기다리는 버튼의 기본 상태입니다.

**호버**: 버튼이 상호작용을 위해 준비되어 있으며 사용자 작업을 기다리는 상태입니다.

**비활성화**: 버튼이 일시적으로 상호작용할 수 없는 상태입니다. 더 나은 사용자 경험을 위해 버튼이 비활성화된 이유를 설명하는 것이 좋습니다. 특히 직관적이지 않은 경우 더욱 그렇습니다. 이 설명은 버튼 옆의 텍스트 또는 호버 시 툴팁으로 제공할 수 있습니다.

**로딩**: 사용자가 시작한 작업에 대한 응답으로 시스템이 작업을 실행 중임을 나타내는 상태입니다.

**선택됨**: 사용자가 여러 옵션 중 하나를 선택할 수 있는 전환 가능한 요소에 사용되는 상태입니다. "선택됨" 상태의 버튼은 현재 선택된 상태 또는 옵션을 나타냅니다.

## 크기

각 버튼에는 네 가지 크기가 있어 컨텍스트에 따라 가장 적합한 옵션을 선택할 수 있습니다.

![Sizes](/static/images/design/Button/Button11.png)

**XS**: 크기가 제한된 구성 요소의 작업에 사용되는 추가 크기입니다.

**S**: 표준 버튼이 너무 크고 테이블이나 작은 카드와 같이 많은 공간을 차지하는 상황에서 사용됩니다.

**M**: 기본적으로 권장되는 주요 크기입니다.

**L**: 생성 양식, 모달 또는 팝업 창에서 주요 작업을 강조하는 데 사용됩니다.

**XL**: 웹사이트 또는 랜딩 페이지에 충분한 공간이 있고 작업을 강조해야 하는 곳에서 사용됩니다.

## 모양

"pin" 속성을 사용하여 버튼 모양을 제어하고 다음 값으로 스타일을 설정할 수 있습니다.

![Form](/static/images/design/Button/Button12.png)

**round**: 둥근 모양의 버튼, 즉 양쪽에 둥근 모서리가 있는 버튼입니다. 이 스타일은 버튼에 부드럽고 현대적인 느낌을 주어 상호작용하기에 친근하고 즐겁습니다.

**brick**: 양쪽에 날카로운 모서리가 있는 버튼입니다.

**clear**: 양쪽에 날카로운 모서리가 있고 윤곽선이 없는 버튼입니다.

**circle**: 양쪽에 원형 모양의 버튼입니다.

이러한 속성을 다양한 변형으로 결합하여 디자인 컨셉 및 인터페이스 요구 사항에 따라 고유한 버튼 스타일을 만들 수도 있습니다.

## 너비

기본 버튼 너비는 텍스트에 자동으로 맞춰지므로 버튼이 텍스트를 모두 수용할 수 있도록 너비가 늘어나 잘리지 않습니다. 그러나 버튼 너비를 제어해야 하는 경우 자동 및 최대의 두 가지 주요 속성을 사용할 수 있습니다.

![Width](/static/images/design/Button/Button13.png)

**auto**: 버튼 치수가 컨테이너 경계를 벗어나지 않도록 제한합니다. 버튼 내용이 컨테이너에 맞지 않으면 줄임표로 숨겨져 추가 텍스트가 있음을 나타냅니다.

**max**: 부모 컨테이너 너비와 같은 고정 버튼 너비를 설정합니다. 버튼 내용이 이 너비를 초과하면 인터페이스를 간결하게 유지하고 컨테이너 오버플로를 방지하기 위해 줄임표로 잘립니다.

### 버튼 텍스트

- 버튼 텍스트는 사용자가 클릭 후 어떤 일이 일어날지 즉시 이해할 수 있도록 명확하고 간결하게 작성하세요.
- 아이콘을 사용할 때는 의미를 명확하게 전달하고 사용자가 쉽게 인식할 수 있도록 하세요.
- 아이콘과 텍스트 간의 균형을 유지하여 두 요소 모두 눈에 잘 띄고 읽기 쉬우며, 어느 한쪽이 지배하지 않도록 하세요.
- 아이콘만 있는 버튼을 사용할 때는 마우스 호버 시 액션 툴팁을 추가하여 사용자가 아이콘의 의미를 볼 수 있도록 하세요.

**텍스트 작성**

- 버튼의 텍스트는 클릭 후 발생할 작업의 핵심을 명확하고 간결하게 전달해야 합니다. 긴 문구나 불분명한 약어는 피하세요.
- 사용자의 행동을 설명하는 동사를 사용하세요. 예를 들어, 저장, 보내기, 삭제 등입니다.
- 버튼에 부정적인 단어나 문구를 사용하지 마세요. 부정적인 지시는 혼란을 야기할 수 있습니다.
- 모든 인터페이스 버튼의 텍스트에 일관된 스타일과 형식을 유지하세요. 이는 사용자가 쉽게 탐색하고 어떤 작업을 사용할 수 있는지 이해하는 데 도움이 됩니다.
