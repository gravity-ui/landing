SegmentedRadioGroup은 라디오 수신기의 밴드 스위치 원리로 작동하는, 행으로 배열된 두 개 이상의 버튼 집합입니다. Radio 스위치와의 혼동을 피하기 위해 Segmented-control이라고도 합니다.

## 언제 사용해야 할까요?

SegmentedRadioGroup은 폼 작성, 목록 또는 테이블 필터링, 페이지 레이아웃 변경, 도구 작동 모드 조정 시 두 개 이상의 옵션 중에서 선택하는 데 사용됩니다. 모든 옵션이 한 번에 표시되므로 드롭다운 메뉴보다 사용자 친화적입니다.

SegmentedRadioGroup 옵션은 문맥적으로 관련이 있고 상호 배타적이어야 합니다.

옵션 수는 2-5개로 제한해야 하며, 옵션 이름은 간결해야 합니다(최대 3단어).

SegmentedRadioGroup을 초기화할 때 기본적으로 하나 이상의 옵션이 선택되어야 합니다.

## 언제 사용하지 않아야 할까요?

여러 옵션을 사용할 수 있는 경우 체크박스 또는 드롭다운 목록(Select)을 사용하세요.

"예 / 아니요" 또는 "켜기 / 끄기"와 같은 옵션의 경우 체크박스 또는 스위치(Switch)를 사용하세요.

옵션이 5-6개 이상이거나 옵션 이름이 긴 경우, 라디오 스위치 그룹(Radio) 또는 드롭다운 목록을 사용하세요.

페이지 탐색 또는 진행률 표시줄(예: 마법사에서)에 SegmentedRadioGroup을 사용하는 것은 허용되지 않습니다. 페이지에서 탐색하려면 탭 그룹(Tabs)을 사용하세요.

### 크기

M은 데스크톱 인터페이스의 표준 크기이며, XL은 모바일 인터페이스의 표준 크기입니다. 페이지 또는 대화 상자의 다른 요소와의 시각적 일관성을 유지하기 위해 다른 크기를 사용할 수도 있습니다.

![Sizes](/static/images/design/SegmentedRadioGroup/Sizes1.png)

버튼 높이를 조정하는 것 외에도 내부 패딩, 버튼 모서리 둥근 정도, 글꼴 크기를 사용자 정의할 수 있습니다.

옵션이 있는 버튼은 너비가 다양할 수 있으며 콘텐츠에 맞게 자동으로 조정될 수 있습니다(Auto width). 또는 모든 버튼의 너비가 같고 컨테이너 너비를 반으로 나눌 수 있습니다(Max width).

![Sizes](/static/images/design/SegmentedRadioGroup/Sizes2.png)

마지막 옵션은 이름이 몇 글자만 다를 때 사용됩니다. 그렇지 않으면 이름이 버튼 가장자리를 벗어날 수 있습니다.

요소 전체가 균형 잡히도록 Auto 모드에서도 SegmentedRadioGroup 내의 버튼 크기를 동일하게 유지하려고 노력하세요.

### 콘텐츠

텍스트(Label) 외에도 버튼에는 아이콘과 카운터가 포함될 수 있습니다. Button 요소와 마찬가지로 아이콘은 텍스트와 함께(Icon and Label) 또는 텍스트 없이(Icon only) 표시될 수 있습니다. 동일한 SegmentedRadioGroup 내에서 다른 유형의 콘텐츠가 있는 버튼을 결합하지 마세요.

Counter는 옵션에 포함된 개체 수를 사용자에게 알립니다.

![Content](/static/images/design/SegmentedRadioGroup/Content.png)

SegmentedRadioGroup 내의 버튼 텍스트는 대문자로 시작해야 합니다. 텍스트는 줄임표로 잘리거나 반복되는 부분이 없어야 합니다.

아이콘만 있는 버전에서는 아이콘 위에 마우스를 올렸을 때 아이콘 이름이 포함된 툴팁(Tooltip)이 표시되어야 합니다.

폼에서는 텍스트만 있거나 텍스트와 아이콘이 있는 옵션을 사용하는 것이 좋습니다. 아이콘만 있는 모드는 일반적으로 레이아웃 간 전환 또는 사용자에게 이미 익숙한 옵션 선택(예: 목록과 개체 썸네일 간 전환 또는 편집기에서 서식 유형 선택)에 사용됩니다.

### 상태

기본 상태(Default) 외에도 버튼은 선택됨(Selected) 및/또는 비활성화됨(Disabled) 상태일 수도 있습니다. 마우스를 올렸을 때(Hover) 버튼도 상태를 표시합니다. 한 번에 하나의 버튼만 선택할 수 있습니다.
