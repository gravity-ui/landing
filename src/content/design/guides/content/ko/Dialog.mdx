**모달 창**은 추가 콘텐츠를 표시하고 특정 작업이나 시나리오에 사용자의 주의를 끌기 위해 사용됩니다. 모달 창은 사용자가 창을 닫거나 지정된 작업을 실행할 때까지 부모 페이지와의 상호 작용을 차단합니다.

### 진입점

모달 창으로 진입하는 방법은 특정 트리거(예: 인터페이스의 버튼)를 클릭하거나, 인터페이스의 특정 작업 또는 조건(예: 저장하지 않고 페이지를 나가려는 시도 또는 다른 탭에서 계정 변경)에 의해 이루어질 수 있습니다.

### 크기

모달 창의 크기는 콘텐츠에 따라 결정됩니다.

![Sizes](/static/images/design/Dialog/sizes.png)

**S 사이즈 - 480px**: 작은 알림/확인 메시지 및 양식에 사용됩니다.

**M 사이즈 – 720px**: 작거나 중간 크기의 양식에 사용됩니다.

**L 사이즈 – 900px**: 많은 양의 데이터 및 모달 창 내에서의 흐름에 사용됩니다.

**커스텀 사이즈**: 세 가지 옵션(S, M, L) 중 어느 것도 적합하지 않을 때 사용됩니다.

### 콘텐츠 영역

모달 창에는 모든 유형의 콘텐츠가 포함될 수 있습니다. 레이아웃은 사용자의 요구 사항에 따라 선택됩니다.

모달 창에 양식이 포함된 경우, 다음 두 가지 요소 레이아웃 중 하나를 권장합니다.

### 버튼

대부분의 경우 작업 버튼이 필요합니다. 사용자가 모달 창에 지정된 작업을 완료하지 않으면 버튼이 비활성화됩니다.
왼쪽에는 사용자 정의 컨트롤 섹션이 있어 보조 버튼과 링크를 배치할 수 있습니다. 예를 들어, 참조 링크 등이 있습니다.

![Group 507124116.png](/static/images/design/Dialog/Group_507124116.png)

경우에 따라 작업 버튼 대신 위험 버튼을 사용할 수 있습니다 (예: 엔티티 삭제 모달 창).

![Group 2087325976.png](/static/images/design/Dialog/Group_2087325976.png)

모달 창이 정보 제공 목적만 있고 대상 작업이 없는 경우 작업 버튼이 필요하지 않습니다. 이 경우 일반 스타일의 닫기 버튼을 사용할 수 있습니다.

![Frame 2087326371.png](/static/images/design/Dialog/Frame_2087326371.png)

보조 버튼은 항상 취소 버튼으로 사용됩니다. 추가 작업(예: 변경 없이 계속 진행)을 수행하는 데 사용할 수 있습니다.

![Group 2087326024.png](/static/images/design/Dialog/Group_2087326024.png)

버튼이 전혀 없을 수도 있습니다. 일반적으로 콘텐츠 영역 자체 내에서 선택이 이루어져야 하며, 그렇지 않으면 부모 인터페이스를 사용할 수 없게 됩니다.

![Frame 2087325516.png](/static/images/design/Dialog/Frame_2087325516.png)

### 모달 내 탐색

모달 창에 여러 단계가 포함된 경우, 뒤로 가기 버튼을 사용하여 이전 단계로 돌아갈 수 있습니다.

![Group 2087326025.png](/static/images/design/Dialog/Group_2087326025.png)

모달 창이 중첩되어 있지만 단계가 선형적이지 않은 경우, 창의 콘텐츠 영역에 브레드크럼을 사용할 수 있습니다.

![breadcrumbs](/static/images/design/Dialog/breadcrumbs.png)

### 페이지 내 모달 창 위치 지정

모달 창은 화면 중앙에 세로 및 가로로 정렬됩니다.

뷰포트 크기를 조정할 때 위치는 변경되지 않으며, 모달 창의 크기는 콘텐츠에 영향을 주지 않는 한 줄어들 수 있습니다.

화면 상단 및 왼쪽 가장자리로부터 최소 20px의 간격을 유지해야 합니다.

![position](/static/images/design/Dialog/position.png)

창이 뷰포트 안에 들어가지 않는 경우, 상단 및 왼쪽 간격은 그대로 유지되고 모달 창의 오른쪽 및 하단 가장자리가 화면 가장자리를 벗어날 수 있습니다. 하지만 이는 권장되지 않습니다. 따라서 모달 창은 뷰포트 안에 들어가고 모든 면에서 화면 가장자리로부터 최소 20px의 간격을 유지하도록 설계해야 합니다.

### 창 닫기

대상 작업 외에도 모달 창은 다음과 같은 방법으로 닫을 수 있습니다:

<div className="row">
   <div className="col-12 col-md-6">
      1. 취소 버튼

      ![cancel_btn](/static/images/design/Dialog/cancel_btn.png)
   </div>
   <div className="col-12 col-md-6">
      2. 'X' 버튼

      ![x_btn](/static/images/design/Dialog/x_btn.png)
   </div>
</div>

모달 창이 아닌 경우 다음 방법 중 하나를 사용하여 닫을 수 있습니다.

- 모달 창 외부 클릭
- Esc 키 누르기

'취소' 및 'X' 버튼은 모달 창에서 널리 사용되는 익숙한 요소입니다. 모달 창 외부를 클릭하면 사용자가 특정 요소에 마우스 커서를 맞출 필요 없이 더 쉽게 창을 닫을 수 있습니다. Esc 키를 누르면 마우스 없이 창을 닫을 수 있습니다.

모달 창과만 상호 작용할 것으로 예상되므로, 부모 페이지의 콘텐츠는 스크롤되지 않아야 합니다.

### 사용자 데이터 손실 방지

모달 창은 쉽게 닫을 수 있으므로 실수로 닫을 가능성이 항상 있습니다. 사용자가 모달 창에서 체크박스만 선택한 경우, 사용자가 상당한 시간을 들여 코드를 작성하거나 12자리 차량 VIN 코드를 다시 입력한 경우보다 손실된 데이터를 복구하기가 더 쉽습니다. 따라서 이러한 상황은 피하는 것이 좋습니다. 그러나 사용자가 일반적인 방법으로 모달 창을 닫는 것을 막아서는 안 됩니다.

**데이터 손실 위험을 피하기 위해 데이터 삭제에 대한 추가 확인 창을 사용하는 것을 권장합니다.** 이 경우 사용자가 작업을 확인해야 합니다.

![Pop-up S.png](/static/images/design/Dialog/Pop-up_S%202.png)

모달 창을 닫을 때, 추가 확인 창이 필요한지 여부를 판단하기 위해 상식을 사용하십시오. 안전과 사용자를 짜증나게 할 수 있는 너무 많은 질문을 하는 것 사이의 균형을 맞추는 것이 중요합니다. 확인의 목적은 사용자가 스크립트를 실행할 수 있도록 하면서도 사용자의 입력을 잃지 않도록 하는 것입니다.
