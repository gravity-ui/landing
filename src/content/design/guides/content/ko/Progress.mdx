이것은 다양한 인터페이스에 가로 막대 차트를 포함하기 위한 컴포넌트입니다. 데이터 분포, 점유율 또는 진행 상황을 표시할 수 있습니다.

## 사용 사례

✅ **언제 사용해야 할까요?**
디스크의 여유 공간, 완료된 작업 수 등과 같은 항목의 완료 또는 채워진 정도를 표시해야 할 때 사용하세요.

🚫 **언제 사용하지 않아야 할까요?**
비교 없이 단일 값을 표시해야 할 때 사용하지 마세요. 예를 들어, 15/60을 표시하려면 Progress를 사용할 수 있지만 15만 표시하는 데는 사용하지 마세요.

## 구조

![구조](/static/images/design/Progress/progress-0001.png)

1. 표시기의 색상. 필수 항목
2. 텍스트. M 사이즈와 함께 사용하며, 선택 항목입니다.
3. 배경. 필수 항목

## 사이즈

![사이즈](/static/images/design/Progress/progress-0002.png)

1. **M**은 텍스트가 있는 유일한 사이즈입니다. 공간이 많은 섹션이나 진행률 표시기가 주요 요소인 페이지에 사용하세요.
2. **S**는 테이블, 카드 및 진행률이 페이지의 유일한 주요 구성 요소가 아닌 개별적인 경우에 사용됩니다.
3. **XS**는 콘텐츠를 간결하게 유지해야 하는 좁은 공간에 사용됩니다.

## 테마

![테마](/static/images/design/Progress/progress-0003.png)

**기본(Default)**은 페이지에 더 중요한 요소가 포함되어 있어 진행률에 특별한 주의를 끌 필요가 없을 때, 중립적인 경우에 가장 자주 사용됩니다.

**정보(Info)**는 진행률에 더 중점을 두거나 작업이 진행 중임을 표시해야 하는 중립적인 경우에도 사용됩니다.

**성공(Success)**은 성공적으로 완료된 프로세스나 시스템이 문제없이 실행되고 있음을 나타내는 데 사용됩니다.

**경고(Warning)**는 특이한 상황에서 사용자에게 무언가를 경고하는 데 사용됩니다. 예를 들어, 임박한 문제에 사용자의 주의를 끌기 위해 사용됩니다.

**위험(Danger)**은 오류가 발생하여 사용자가 문제를 해결하기 위한 조치를 취해야 할 때 최대 주의를 끌기 위해 사용됩니다. 예를 들어, 디스크 공간 부족, 시스템 오류 등이 발생했을 때 사용됩니다.

**기타(Misc)**는 기본(Default)을 사용하고 싶지 않은 중립적인 경우에 사용됩니다.

## 범위

1. **진행률 값에 따라 색상 변경**. 예를 들어, 표준 표시는 파란색(정보)이고 100%는 녹색(성공)인 것처럼, 진행률 값에 따라 색상이 변경되도록 이 컴포넌트를 구성할 수 있습니다.

   ![범위](/static/images/design/Progress/progress-0004.png)

2. **로딩 애니메이션**. 파일 업로드와 같이 사용자에게 실시간 진행 상황을 보여주는 데 사용하세요.

   ![범위](/static/images/design/Progress/progress-0005.png)

3. **사용자 지정 색상**. 다른 모든 옵션이 적합하지 않을 때 Progress 색상 세트를 사용할 수 있습니다. 사용자 지정 색상을 사용하고 여러 색상을 조합하세요.

   ![범위](/static/images/design/Progress/progress-0006.png)
