Toaster는 중요한 정보나 경고를 제공하기 위해 화면에 나타나는 작은 알림입니다. 폼 제출이나 파일 다운로드와 같이 특정 이벤트나 액션에 대해 사용자에게 알리는 데 사용할 수 있습니다.

### 주요 유형

**기본**

즉각적인 조치가 필요 없거나 메시지가 특정 액션 또는 이벤트와 관련이 없을 때 사용됩니다.

![Default](/static/images/design/Toaster/Default.png)

**정보**

프로세스, 이벤트 또는 액션에 대해 사용자에게 알리는 데 사용됩니다. 예를 들어, 데이터 전송 성공 또는 페이지 로드 시에 사용할 수 있습니다.

![Info](/static/images/design/Toaster/Info.png)

**성공**

작업 또는 액션이 성공적으로 완료되었음을 나타냅니다. 예를 들어, 데이터가 성공적으로 저장되었거나 사용자가 인증되었을 때 사용할 수 있습니다.

![Success](/static/images/design/Toaster/Success.png)

**경고**

잠재적인 문제나 오류에 대해 사용자에게 알립니다. 예를 들어, 폼에서 필수 필드를 채워야 하거나 인터넷 연결이 없을 때 사용할 수 있습니다.

![Warning](/static/images/design/Toaster/Warning.png)

**오류**

오류를 나타내며 사용자의 주의가 필요함을 알립니다. 예를 들어, 데이터가 잘못 입력되었거나 서버 오류가 발생했을 때 사용할 수 있습니다. 이 상태는 오류를 수정하거나 더 많은 정보를 얻을 수 있는 액션 버튼과 함께 사용하는 것이 가장 좋습니다.

![Error](/static/images/design/Toaster/Error.png)

### 수정

Toast는 다섯 가지 주요 구성 요소로 이루어집니다.

![Modifications](/static/images/design/Toaster/Modifications.png)

1.  **Toast 유형 아이콘:** 아이콘을 사용하거나 사용하지 않고 구성 요소를 사용할 수 있습니다. 아이콘은 선택적 요소입니다.
2.  **헤더:** 수행 중인 액션에 대해 사용자에게 알립니다. 설명이 포함된 경우 헤더 사용은 선택 사항입니다.
3.  **설명:** 제목을 명확히 하고 독립적으로 사용할 수 있습니다.
4.  **버튼:** Toast는 하나 또는 두 개의 버튼을 가질 수 있으며, 첫 번째 버튼은 액션 버튼으로 사용하는 것이 좋습니다. 버튼은 선택적 요소입니다.
5.  **닫기 아이콘:** 닫기 아이콘은 선택적 요소입니다. Toast에 숨김 기간이 설정되어 있지 않은 경우 닫기 버튼이 필수입니다.

### 사용 시기

Toast는 다음과 같은 시나리오에서 사용하는 것이 좋습니다.

*   폼을 제출하거나 액션을 수행할 때 사용자가 진행 상황을 확인하도록 하고 싶을 때. 예를 들어, 데이터가 성공적으로 전송되었거나 오류가 발생했다는 메시지를 표시할 때.
*   사용자의 워크플로우를 방해하거나 화면을 차단하고 싶지 않을 때. Toast는 화면 모서리에 표시되며 전체 화면을 가리지 않아 사용자가 인터페이스를 계속 사용할 수 있습니다.
*   특정 액션에 대한 추가 정보가 필요할 때. Toast에는 사용자가 무슨 일이 일어났는지 이해하는 데 도움이 되는 추가 콘텐츠나 정보에 대한 링크가 포함될 수 있습니다.

### 사용하지 않을 때

Toast는 다음과 같은 시나리오에서 사용하지 않는 것이 좋습니다.

*   사용자의 즉각적인 응답이 필요할 때.
*   Toast를 표시하는 시점에 수행되는 액션이 문제나 원치 않는 결과를 초래할 수 있을 때.
*   메시지에 많은 정보나 더 자세한 설명이 필요할 때.
*   정보가 이미 화면의 다른 형태로 표시되어 있을 때.

### 화면에서의 위치

Toast는 오른쪽 하단 모서리에 표시됩니다.

외부 여백은 다음과 같습니다. 하단 M (8px), 오른쪽 XL (20px).
