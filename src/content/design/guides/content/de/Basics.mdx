Hier ist die Übersetzung der Design-Guide-Quelldatei ins Deutsche, wobei darauf geachtet wurde, dass sie natürlich klingt und technische Begriffe sowie MDX-Komponenten beibehalten werden:

Gravity-UI ist eine Sammlung von Bibliotheken und Komponenten zum Erstellen funktionaler Benutzeroberflächen. Das Designsystem unterstützt mehrere Themes: **hell**, **dunkel** und **hoher Kontrast** Variationen von jedem. Der visuelle Stil der Komponenten in jedem Theme kann einfach angepasst werden.

**Farbe**

Das Designsystem besteht aus vier Themes, die auf einem zweischichtigen Farbmodell basieren. Die Theming-Engine unterstützt vielseitige Farbanpassungen und Markenidentitäten mit unterschiedlichen Anforderungssätzen.

**Typografie**

Das Typografiesystem besteht aus Schriftschnitten, Display-Schriften und Code (Monospace-Schrift). Sie können die Standard-Schriften verwenden oder Ihre eigenen erstellen.

**Eckenradius von Steuerelementen**

Der Eckenradius ist ein kleiner, aber wichtiger Teil des visuellen Stils. Um die Arbeit damit zu erleichtern, haben wir eine Reihe von Radiusvariablen entwickelt. Diese Variablen werden von verschiedenen Steuerelementen verwendet. Um den Radius der Steuerelemente anzupassen, ändern Sie einfach die Werte der Variablen.

**Erstellen und Verwenden eines Themes**

Unabhängig von der Marke und der visuellen Sprache läuft alles auf eine Reihe von Konstanten hinaus, die Sie einfach bearbeiten und sowohl in Ihrem Code als auch in der Figma-Bibliothek verwenden können.
