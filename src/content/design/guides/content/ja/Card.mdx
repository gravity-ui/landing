Card コンポーネントは、コンテンツを意味のあるブロックに分割するために使用されます。本質的には、コンテンツをグループ化するためのコンテナであり、折りたたむオプションもあります。

## 使用方法

このコンポーネントには4つのタイプがあります。

![Usage](/static/images/design/Card/Placement.png)

## **テーマ付き**

![Placement](/static/images/design/Card/Placement1.png)

テーマ付きのカードは、背景色または境界線の色を変更できます。多くの場合、ブロックのエラー/成功、またはユーザーへの警告を説明するために使用されます。このカードは、デフォルトの色（ノーマル）でコンテンツを組み合わせるために最も一般的に使用されます。`theme` パラメータはカードのテーマを指定し、`View` パラメータはカードが境界線付きか塗りつぶしかを決定します。

このタイプのカード（コンテナ）には、以下のオプションがあります。

**View**: Outlined / Filled

**Theme**: Normal, Info, Success, Warning, Danger (color)

![theme](/static/images/design/Card/theme.png)

- Normal - ほとんどの場合に使用される、境界線付きのカードの基本状態です。
- Info — ユーザーに情報を提供するために使用されます。
- Success — ポジティブな結果のイベントを示すために使用されます。
- Warning — 警告を表示するために使用されます。
- Danger は、ユーザーの最も注意を引くように設計されたコンテナです。データ損失、サービスの中断などによってトリガーされる可能性があります。

## **選択可能なカード**

![Selectable](/static/images/design/Card/Selectable.png)

このカテゴリのカードでは、利用可能なオプションから1つを選択する必要があります。カードには、ノーマル、選択済み、ホバーの3つの状態があります。

このタイプのカード（`type = selection`）には、以下のオプションがあります。

**selected**: true / false

**disabled**: true / false

## **引き上げられた（Raised）**

テーマ付きグループの**Normal**と同様に、コンテンツをグループ化するために使用されます。主な違いは、このカードタイプには影があることです。ウィジェットなど、カードを強調表示する必要がある状況での使用をお勧めします。

![Raised](/static/images/design/Card/Raised.png)

影が隣接するカードにかからないように、これらのカードの間には十分なスペースを空けてください。

Lサイズでは、カードはより「ふんわりとした影」を持つため、このサイズを使用する場合は、Mサイズよりもカード間の間隔をわずかに大きくすることをお勧めします。

**type**: container;

**view**: raised

## **影付きアクション（Raised with action）**

前のカードと同様ですが、カードをクリックする必要があります。**Raised**カードとは異なり、ホバー状態はありますが、選択可能なカードのような「選択オプション」状態はありません。詳細な説明を含む選択肢をユーザーに提供する必要がある場合に使用することをお勧めします。

クリックするとネストされたページにリダイレクトされることに注意してください。したがって、ユーザーはこのカードの動作を期待する必要があります。（カードのデザインから明確であるべきです）

![WithActions](/static/images/design/Card/WithActions.png)

このタイプのカード（`type = action`）には、以下のオプションがあります。

**disabled**: true / false

## サイズ

![Sizes M and L, using the example of cards with a shadow (Raised)](/static/images/design/Card/Size.png)

影付きカード（Raised）の例を使用したMおよびLサイズ

MとLの2つのサイズが利用可能です。角の半径はサイズによって異なります。Lサイズは半径8px、2XLサイズは半径16pxです。サイズ選択は、ページ上の配置、パディング、およびページコントロールのサイズによって異なります。たとえば、ページで入力フィールド、ボタン、その他の要素にLサイズを使用している場合、カードもLサイズにする必要があります。

コンポーネントの幅と高さは、コンテンツによって決まります。最小サイズは、コンテンツからわずかなパディングを付けて選択できます。最大サイズは、ページ幅の100%にすることができます。
