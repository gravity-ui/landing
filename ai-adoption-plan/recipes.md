# Раздел /recipes/ — канонические сценарии

**Приоритет:** P2 · **Зависимости:** нет · **Трекер:** DATAUI-3745, EPIC 4 (задача 4.2); в комментариях отмечено «очень надо даже без LLM»

## Цель

Готовые проверенные рецепты типовых сценариев. Для агентов это самый дешёвый путь к рабочему коду (fetch целиком → адаптировать), для людей — недостающий getting started.

## Шаги

1. Стартовый набор (≥5, по эпику; Камендровский: «мб даже больше 5 пригодится»):
   - Vite SPA: установка, стили, ThemeProvider, переключатель темы;
   - Next.js (App Router): SSR-нюансы, стили, тема;
   - Дашборд: layout + Card + Table + charts, синхронизация темы charts ↔ uikit;
   - Форма: TextInput/Select/валидация (+ когда брать dynamic-forms, а когда — нет);
   - App shell: AsideHeader/navigation (агентов «приятно удивляет» его существование — значит, сами не находят);
   - Кандидаты следом: i18n, кастомная тема через uikit-themer, иконки.
2. Формат рецепта: одна страница = один сценарий; полный рабочий код без сокращений; версии пакетов во фронтматтере; частые ошибки в конце. Каждый рецепт доступен как `.md` (md-endpoints.md).
3. Верификация: код каждого рецепта — компилируемый проект в e2e/CI (рецепт, который не собирается, хуже отсутствующего: агент скопирует его с полным доверием).
4. Индексация: секция Recipes в корневом llms.txt (root-llms-txt.md), перекрёстные ссылки из пакетных llms.txt и managed-секции AGENTS.md.
5. Использовать промпты из evals-baseline.md как приёмку: рецепты должны закрывать самые частые сценарии из набора.

## Definition of Done

- [ ] ≥5 рецептов опубликованы на сайте и доступны как .md.
- [ ] Код рецептов собирается в CI.
- [ ] Рецепты в корневом llms.txt.
- [ ] Evals-прогон по сценарным промптам улучшился против baseline.
