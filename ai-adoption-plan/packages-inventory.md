# Инвентаризация публичных пакетов

**Приоритет:** P0 · **Зависимости:** нет · **Трекер:** [DATAUI-3747](https://st.yandex-team.ru/DATAUI-3747) · **Исполнитель:** dagaponov

## Цель

Единый источник истины по всем публичным пакетам `@gravity-ui/*`: описания, обвязка, peer-зависимости. Его читают генераторы llms.txt, managed-секции AGENTS.md и CI-проверки.

## Решение

По итогам обсуждения в тикете (комментарий от 14 мая): **никаких отдельных yaml-файлов не заводим**. Курируемый список — существующий `src/libs.mjs`, недостающая информация дозаполняется автоматически из README и npm **на этапе генерации**, без промежуточных персистентных артефактов. README — источник истины; для машиночитаемых блоков договариваемся о структуре README (readme-template.md). Ценные практики из исходной постановки сохраняем: trip-wire, чек-лист описаний, blind-test, rationale.

## Шаги

1. Расширить записи в `src/libs.mjs` полями, которые нельзя достать из README/npm:
   - `llmDescription` — одно предложение (EN), отвечает на вопрос «когда взять этот пакет, а не соседний»;
   - `storybookUrl`, `docsUrl` (если ещё нет).
     Required setup **не дублируем в libs.mjs** — он обязан присутствовать в README (секция по шаблону из readme-template.md), генераторы забирают его оттуда.
2. Дозаполнение на этапе генерации (внутри пайплайна `generate-llms.mjs`, в памяти): peer-зависимости и версии — из npm registry / package.json пакета; наличие README компонентов — обходом репозитория; мета-секции — парсингом README по договорённым маркерам.
3. Trip-wire в CI (скрипт сверяет npm registry с `libs.mjs`, без промежуточных файлов):
   - новый пакет `@gravity-ui/*` в npm без записи в `libs.mjs` → красная сборка / автосозданный issue с шаблоном записи;
   - пакет в `libs.mjs`, отсутствующий в npm (depublished/переименован) → алёрт;
   - у Tier-1/Tier-2 записи пустой `llmDescription` или в README пакета нет обязательных мета-блоков → красная сборка.
4. Написать `llmDescription` для 8 Tier-1 пакетов (uikit, charts, navigation, icons, date-components, table, markdown-editor, page-constructor) по чек-листу из тикета: одно предложение; различает от соседей; без маркетинга; понятно без имени; не дублирует соседа по ключевым словам. Эти же формулировки уезжают в README пакетов (позиционирование для людей и LLM едино — пункт 1 твоего комментария).
5. Blind-test болевых пар силами человека вне задачи: charts/chartkit/yagr/graph; date-components/date-utils; page-constructor/page-constructor-builder/blog-constructor; dialog-fields/dynamic-forms; Table (uikit) / @gravity-ui/table. Неуверенное угадывание → переписать и повторить.
6. `data/llms/descriptions-rationale.md`: для каждого Tier-1 — финальное описание, чем отличается от соседей, почему такая формулировка.

## Definition of Done

- [ ] Все ≥24 пакета имеют `llmDescription` и `docsUrl` в `libs.mjs`.
- [ ] Пайплайн генерации дозаполняет peer-deps и мета-секции из npm/README без персистентных промежуточных файлов.
- [ ] Trip-wire фейлит сборку по всем трём правилам (проверено на синтетическом кейсе).
- [ ] Tier-1 описания прошли чек-лист и blind-test, rationale опубликован.
- [ ] Правило обновления зафиксировано в CONTRIBUTING.md.
