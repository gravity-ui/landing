# Аналитика AI-эндпоинтов

**Приоритет:** P2 · **Зависимости:** нет (расширяется по мере появления эндпоинтов) · **Трекер:** DATAUI-3745, EPIC 1

## Цель

Видеть, кто и что реально забирает: без этого невозможно понять, работает ли «сайтовый» слой, и подтвердить/опровергнуть выводы о discovery на собственных данных.

## Шаги

1. Логирование запросов к `/llms.txt`, `/<package>/llms.txt`, `/llms/packages.yaml`, `*.md`-эндпоинтам, `/recipes/*`: user-agent, referer, `Accept`-заголовок, IP-класс (датацентр/резидентный).
2. Классификация клиентов по user-agent: Claude-Code / Claude-User / ClaudeBot / GPTBot / ChatGPT-User / cursor / PerplexityBot / браузеры / SEO-краулеры. Отдельно считать запросы с `Accept: text/markdown` на HTML-URL (сигнал агентского трафика без суффикса).
3. Дашборд (внутренний): запросы по эндпоинтам × клиентам × времени; топ забираемых пакетных файлов и рецептов.
4. Через 4–6 недель после запуска пакетных llms.txt — отчёт в эпик: каким файлами пользуются, какими нет; решения о детализации/бюджетах на этих данных.
5. Бенчмарк-ориентир: у Mintlify-хостящихся доков медиана — 14 обращений/мес к llms.txt и 79 к llms-full.txt, почти всё — user-directed. Если у нас сопоставимо мало — это норма канала, а не провал; ценность смотреть в связке с evals и adoption docs-in-npm.

## Definition of Done

- [ ] Логи собираются и классифицируются по клиентам.
- [ ] Дашборд доступен команде.
- [ ] Первый отчёт с выводами опубликован в DATAUI-3745.
