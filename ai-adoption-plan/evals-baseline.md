# Baseline-замер и regression-набор промптов

**Приоритет:** P0 · **Зависимости:** нет · **Трекер:** [DATAUI-3746](https://st.yandex-team.ru/DATAUI-3746) (в работе у iakimovaleks) — эта карточка дополняет её выводами анализа

## Цель

Мерить эффект каждого изменения, а не верить в него. Замер «как сейчас» — до мержа любых задач из этого плана.

## Контекст

Внутренняя метрика из саммари встречи 25.06 — «сокращение пути агента» (токены, число прочитанных файлов/директорий). Она нужна, но недостаточна: короткий путь к неправильному коду — тоже короткий. Референс: [nextjs.org/evals](https://nextjs.org/evals) (Vercel меряет task success на API после training cutoff; их evals показали 53% baseline / 100% с индексом доков в AGENTS.md и привели к смене дефолтов в create-next-app).

## Шаги

1. Набор из 15–25 промптов, покрывающий:
   - типовые сценарии: «SPA на Vite с переключателем темы», «дашборд с Table + Card + charts», «форма с валидацией», «навигация с AsideHeader»;
   - болевые пары маршрутизации: задачи, где правильный ответ — charts, а не chartkit; date-components, а не date-utils; @gravity-ui/table, а не Table из uikit;
   - **API после training cutoff моделей** — компоненты/пропсы из свежих релизов (например, из changelog uikit за последние 2–3 месяца): именно здесь виден эффект документации, а не памяти модели.
2. Матрица прогонов: {Claude Code, Cursor, Codex CLI} × {без подсказок; с llms.txt в контексте; с доками в node_modules + AGENTS.md-секцией (после docs-in-npm.md)}.
3. Метрики на прогон: компилируется/типчекается (tsc — бесплатный судья); использованы существующие API (нет выдуманных пропсов); выбран правильный пакет; токены и число прочитанных файлов; количество итераций до рабочего результата.
4. Автоматизация: скрипт прогона (можно на Claude Agent SDK / headless-режимах CLI), результаты — в json + сводный markdown. Прогон повторяется после каждого значимого мержа из дорожек A/B.
5. Зафиксировать baseline-отчёт до старта остальных задач.

## Definition of Done

- [ ] Regression-набор закоммичен (промпты + ожидания + скрипт прогона).
- [ ] Baseline снят минимум на двух агентах, отчёт опубликован в тикете DATAUI-3746.
- [ ] Определён порог «изменение принято» (например, +N п.п. executable-rate на post-cutoff задачах).
- [ ] Прогон воспроизводим одной командой.
