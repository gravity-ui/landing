# Context7: claim, конфиг, автообновление

**Приоритет:** P0 (быстрая победа, ~день работы) · **Зависимости:** нет · **Трекер:** нет карточки — завести под DATAUI-3745

## Цель

Закрепить контроль над уже существующей индексацией Gravity UI в Context7 — MCP-сервере документации, который уже подключён у большой доли пользователей Cursor/Claude Code (~58k звёзд, топ инсталляций на Smithery).

## Контекст

Экосистема проиндексирована автоматически и без нашего участия: `/gravity-ui/uikit` — 1155 сниппетов, trust score 9.3, verified; также charts, table, navigation, aikit, page-constructor, icons, markdown-editor и др. Библиотеки не «claimed», конфига нет — качество и свежесть индексации нам не подконтрольны. Есть паразитная запись `/llmstxt/gravity-ui_llms_txt` (trust 6.7) и зеркало с trust 7.0, которые размывают выдачу.

## Шаги

1. Claim всех библиотек организации на [context7.com](https://context7.com) (через GitHub-аккаунт органзации) — даёт админку и повышенные лимиты переиндексации. Инструкция: [context7.com/docs/adding-libraries](https://context7.com/docs/adding-libraries).
2. Добавить `context7.json` в Tier-1 репозитории: правильные folders/exclude (не индексировать changelog-шум, internal-доки), rules с обязательной обвязкой (импорты стилей, ThemeProvider) — они показываются агенту вместе со сниппетами.
3. GitHub Action переиндексации на релиз (тег/publish) в Tier-1 репах.
4. Проверить выдачу до/после на болевых парах (запрос «gravity ui chart» должен вести в charts, не в chartkit) и на post-cutoff API из evals-набора.
5. Зафиксировать владение: кто в команде админ, куда смотрят алёрты.
